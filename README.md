# git
### 1. git reflog
>记录在这个git仓库里面的每一次操作，基本的形式就是:提交的索引号，相对于HEAD指针的位置，操作的命令以及commit时候的信息

### 2. git checkout -- 撤销工作目录中的修改。前提是这个文件还没有被添加到暂存区。如果被添加到暂存区，则可以先使用git reset HEAD命令
		将暂存区的文件，撤销到工作区，然后再使用这个命令撤销修改
>注意：git checkout . 是将工作目录所有文件的修改，都进行撤销。git checkout [filename] 是将制定的filename文件进行撤销修改

### 3. git rm [filename] -- 从git仓库中删除指定名字的文件。
>这个命令会将工作目录中的文件删除，有的时候会提示使用-f选项，强制的删除制定的文件

### 4. git remote add -- 使本地的仓库和远程仓库进行关联。
>关联的基本格式就是：git remote add [选项] name url。其中：name这个选项要和远程仓库分支的名称相同（不然就是添加远程仓库）。通过url将本地的分支和远程仓库的分支建立起关联之后，当我们通过git push origin 远程仓库分支的名称。才能够将我们的代码push到远程分支上，用我们自己随意取的是不能push上去的。

### 5. git clone url -- 从远程仓库copy一份到本地仓库

### 6. git push name branchname -- >将本地的修改push到远程仓库里面去，name指的是远程仓库的名称，默认是origin。branchname指的是远程仓库中的分支的名称，默认值master(这两个名称在我们创建好远程仓库的时候自动的设置好了，我们clone本地的时候，也会有一个master分支，同时和远程的master分支做关联)

### 7. git branch [branch-name] -- 这条命令主要是用于创建分支的
>分两种情况使用，如果没有指定分支的名称，那么，则是查看本地有多少分支以及当前处于哪个分支上。如果指定了分支的名称，则是创建一个新的本地分支

### 8. git checkout <branch-name> 
>这个命令主要是用于切换到指定的分支的。这个命令的后面如果跟着的是文件的名称，则是用于撤销修改的。

### 9. git checkout -b [branch-name]
>这个命令主要是用于创建并切换分支的，这个命令其实是git branch [branch-name]和git checkout [branch-name]的综合使用

### 10. git branch -- 用于查看当前的分支，并同时列出所有的分支
>指定参数-a可以查看远程的分支的信息

### 11. git merge --no-ff [branch-name] -- 合并指定分支到当前的分支
>如果你只有一个commit，可以使用 git cherry-pick [分支名，或者commitID]

### 12. git branch -d <branch-name>:删除本地的分支
>注意，如果你正在删除的分支还没有被其它分支合并过，那么我们可以运行git branch -D 
<branch-name>来强制的删除分支

### 13. git push origin :[远程分支的名称]：这个命令主要是用于删除远程分支
>注意，远程分支的名称之前有一个空格和冒号。

### 14. git push origin [本地分支的名称]:[远程分支的名称]：将本地仓库推送到远程仓库上去

##git的一些高级用法
### 1. git [命令的名称] --help
>这个命令主要是用于获取一条命令的具体的用法

### 2. git blame [文件的名称]
>查看指定文件的每一行的最后一次被更新的信息

### 3. git remote add [仓库的名称] [仓库的地址]
>这条命令主要是用于添加一个远程仓库，添加好之后，我们就可以使用git fetch [仓库的名称]来追踪远程仓库了

### 4. git remote show：列出所有的远程仓库的名称

### 5. git remote rename [旧仓库的名称] [新仓库的名称]
>这条命令主要是给远程仓库更改名称

### 6. git remote remove [仓库名称]
>这条命令主要是用于删除远端的仓库。这个操作除了会删除与这个仓库所有的分支和配置信息

### 7. git remote show [仓库的名称]：这个命令主要是用于列出远程仓库
>如果没有指定远程仓库的名称，则会列出所有的远程仓库的名称，如果，指定了某一个远程仓库的名字，那么则会给出这个远程仓库的相关信息以及配置信息

### 8. git remote set-url [仓库名称] [新仓库地址]：这个命令主要是用于更新特定远程仓库下面的部分仓库地址
>其中的仓库地址可以是正则表达式

### 9. git remote set-url --delete [仓库名称] [仓库地址]--(8和9暂时不理解)

### 10. git branch [branchname] [commitID]：基于当前分支的某次ID来创建一个新的分支
>这个命令所创建的分支的工作区和指定的commitID时候的工作区的状态是一样的

### 11. git branch (-m | -M) [旧分支的名称] [新分支的名称]
>这个命令主要用于修改分支的名称。如果你所修改的分支指定的名称已经存在的话，-m是不会修改分支的名称的，同时会给出提示。-M则会强制的覆盖已经有的分支

### 12. git branch (-d | -D) [分支的名称]
>这个命令主要是用于删除分支的，如果你删除的分支还没有被合并，那么使用-d选项的时候回给出提示，并且不会删除分支，这个时候你可以采取先合并分支，然后再使用-d选项删除分支。或者直接使用-D强制的删除分支。(在进行删除分支之前，不要忘记commit，否则你所修改的内容会被切换的分支的内容覆盖,commit之后再运行删除分支的命令，会出现上面的两种情况);

### 13. git branch -v
>查看每个分支的最后一次提交对象的信息。主要包括commitId，当前分支的状态，以及commit时候的注释信息

### 14. git branch --merged
>这个命令主要是用于查看哪些分支已经并入到了当前的分支里面。这个命令展示出来的分支，都是一些我们之前已经合并到当前分支的分支。这个向我们传达了一个信息，所展示出来的分支都已经合并到当前分支了，那么，除当前分支外，所有的分支都是可以通过git branch -d的命令进行删除。因为他们所包含的工作已经全部的合并到了其他的分支里面。删除也不会有什么影响。

### 15. git branch --no-merged
>这个命令主要是用于查看在这个git仓库里面没有合并到当前分支的分支.(注意，对于刚刚建立的分支，运行这个命令你是看不到效果的。只有更改了一部分的内容并提交之后才能看到效果).

## stash：主要用于暂存当前的工作状态
### 1. git stash [save [message]
>主要用于储存当前的工作状态，同时，我们可以添加一些必要的说明，和commit的信息一样，便于后期的查找.(缓存的前提就是当前的工作目录不是clean的，不然是不能够进行暂存的)

### 2. git stash list
>这个命令主要是用于列出所有的暂存。列出的信息主要包括暂存的索引号，暂存的分支，以及暂存的说明性信息

### 3. git stash show [stash]
>这个命令主要是用于显示某个特定的暂存和暂存的起始点之间内容的变化，类似于diff命令，默认是最后一次的暂存，我们也可以指定特定的暂存。

### 4. git stash pop [stash]
>恢复暂定的缓存，同时从暂存列表移除。如果恢复的时候发生了冲突，则暂存是不会从列表中移除的。这个时候就需要我们手动的结局冲突之后，使用git stash drop手动的将暂存信息从列表中移除.(注意处理完冲突之后，不要忘记commit，否则，下次你是不不能够使用stash添加暂存的)(事实上，我们随时可以使用git stash drop删除暂存信息，如果，我们在pop或者apply之前drop的话，我们是无法通过上述的两个命令恢复特定的暂存的)

### 5. git stash apply [stash]
>这个命令和pop相似，主要是用于恢复特定的暂存的。但是。这个命令是不会自动的删除暂存列表中的暂存信息的。

### 6. git stash drop[stash]
>这个命令主要是从暂存列表中删除特定暂存的。如果没有参数，则默认是最后一次的暂存

### 7. git stash clear
>这个命令是一次性的移除暂存列表中的所有的暂存

##diff
>在了解diff命令之前，我们需要知道，对于git所管理的文件主要有三种状态：工作区的文件，已经添加到暂存区的暂存文件，还有就是已经提交到本地git仓库但还没有被push到远程仓库的文件。

### git diff [文件的名称]
>这个命令主要是用于查看工作目录和暂存区文件的差别的

### git diff --cached [文件名称]
>这个命令主要是用于查看暂存区的文件和已提交的文件的区别(特殊颜色标明的是暂存区的文件状态)

### git diff HEAD [文件名称]
>主要用于查看工作目录中的文件和已提交文件的区别(特殊颜色标明的是工作目录的文件状态)

>还有一点需要注意的就是，如果上面的三条命令没有指定具体的文件的名称，那么则是比较除了.gitignore文件之外所有文件的区别

### git diff [commitId]/[branch]
>这个主要比较的是当前的工作目录和某一次的某一个分支提价时候的差别

##reset
### git reset -hard HEAD
>回退到一个特定的历史版本。丢掉这次提交之后的所有变更。也就是说这个命令会使你的工作区的文件，回退到，你最近一次commit时候的文件的状态。这个命令一定要小心使用，因为它会强制删除你工作区间的所有变更。

### git reset HEAD--取消add
>这个命令主要是回退到一个特定的版本，这个命令简单的说就是将已经添加到暂存区的文件给撤销回来，这样我们需要重新的使用git add讲文件添加到暂存区

### git reset --soft HEAD--取消commit
>这个命令其实就是将已经commit的最近的一次提交给撤销到暂存区，这时，我们需要重新的commit，才能将文件添加到本地仓库。对于这个命令只是变化的是commit之后的文件，本质上改变的是HEAD的指向。所以，如果我们在commit之后，运行git reset --soft HEAD 命令其实是没有任何作用的。因为这条命令就是告诉git讲我本地仓库指向HEAD指针所指向的地方，事实上，我们commit之后，它已经是指在这个地方了，所以我们看不到任何的效果。想让commit之后，仓库的文件会退到摸个指定的commit，我们可以运行git reset --soft HEAD~n.其中n可以是1-12的任意一个数字，这个命令是将commit会退到n次commit前的版本
